# post-type: show-and-tell

「作ってみた」「試してみた」系のお披露目投稿。**完成物 / 試行が主役**。

## いつ使うか

- 完成した小さなプロダクト / プロトタイプを見せたいとき
- 試した結果（実験・検証）を共有したいとき
- スクリーンショットや動画が主役で、文面は説明補助の場合

## 典型構成

### フック（1 行目）

候補:

- **「X を作ってみた」**: 「スマートホーム連携の bot を試作してみました」
- **「X を試したら Y だった」**: 「Claude Code Hooks で自動 lint を試した
  ら、PR 数が減りました」
- **具体名 + 状態**: 「buzzounce 0.1.0 で出来上がった最初の自動生成
  ツイートがこちらです」

### 本文（中央）

何を作った / 試した / 何が分かったか を **1 点に絞る**。3 つあるなら
箇条書きで（mode 許容範囲内）。

成果物の細部の話は控えめに。「見てもらう」ことに集中する。

### CTA / リンク

- リポ / Demo / 動画リンクのいずれか 1 個
- 添付素材で全部見せる場合はリンクなしでも成立する

## 推奨文字数

- 全体: 140-280 字
- 短い方が効く（添付素材で語る）

## 典型添付パターン

| 添付 | 文面の役割 |
|---|---|
| 1 枚（完成スクショ） | 1 点に絞った語り |
| 2-4 枚（プロセス + 完成 等） | プロセスの 1 場面に絞る |
| 動画（操作デモ）| 「最後の動きが肝」等の hint |

## 例文

### 例 1: understated × oss-dev × 添付 1 枚

> Claude Code 用のプラグイン buzzounce、初期版が形になりました。
> persona / mode / post-type を選んで、画像 1 枚目に何があるかも聞かれて
> から、文面候補が 3 案返ってきます。
>
> https://example.com/your-project/your-repo

### 例 2: understated × oss-dev × 添付動画

> design-director でブリーフから 3 案生成 → 採用 → React JSX として納品
> までの流れを 30 秒に圧縮してみました。
> 最後の納品まで一回も「次のコマンドは何？」を聞かれないのがポイント
> です。

→ 動画が主役。文面では「何に注目するか」のヒントだけ。

### 例 3: casual × general × 添付 4 枚

> AI で個人 OSS のヒーロー画像を生成 → SPI として GitHub に登録 →
> README にも埋め込み、まで自動化してみました ✨

→ 4 枚で各ステップを見せる。文面は流れの 1 行サマリのみ。

## NG 構成

- 「神ツール作りました！」（自己評価語）
- 完成物の説明に文字を多く割く（→ 添付で見せた方が早い）
- 「ぜひ使ってみてください！！！」（過剰宣伝）
- リンクが何のリンクか不明（"こちら" だけ）

## show-and-tell type で迷ったら

- **見てもらいたい** が主役か（→ show-and-tell）
- **使ってもらいたい** が主役か（→ release）
- **教訓を伝えたい** が主役か（→ lesson）
- **テクを伝えたい** が主役か（→ tip）

迷う時は添付素材を撮ってみて、「これだけで伝わるか？」を判断。伝わるなら
show-and-tell、伝わらない / 補足が要るなら release / tip / lesson に倒す。
