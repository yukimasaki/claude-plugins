# post-type: release

新 OSS / 新バージョン / 新機能リリースの告知。**何が新しいか + 1 行価値
訴求 + リンク** が骨格。

## いつ使うか

- 新規 OSS / プラグインを公開したとき
- 新バージョン（メジャー / マイナー / パッチ問わず）をリリースしたとき
- 既存ツールに大きな機能追加をしたとき
- マイルストーン（100 commit / 1 年継続 等）を区切りとして共有したいとき

## 典型構成

### フック（1 行目）

候補:

- **具体名 + 動詞**: 「claude-plugins v0.2.0 を公開しました」
- **数字フック**: 「3 つ目のプラグインを追加しました」
- **逆説 / 自虐**: 「車輪を再発明しました」

夫れ以外（疑問形・煽り）は使わない。

### 本文（中央）

**何が新しいか / なぜ作ったか / どう使うか** のいずれか **1 点に絞る**。

3 点書きたい場合は箇条書き化（mode で許容されている範囲で）。
oss-dev mode では文章型を推奨、general mode では箇条書きも OK。

### CTA / リンク

- リンク先で何が見られるか一行で示す
- 「詳細はこちら」だけでは弱い → 「全プロンプトログを公開」「リリース
  ノート参照」等
- リンク 1 個まで（複数あると distract）

## 推奨文字数

- 全体: 140-280 字
- 短版（140 字）と長版（280 字）の両方作って選ばせると良い

## 典型添付パターン

→ `attachments.md` 詳述

| 添付 | 文面の役割 |
|---|---|
| なし | 機能訴求を文面で全部 |
| 1 枚（hero / SPI 画像）| ストーリー or 機能の 1 点訴求 |
| 2-4 枚（hero + UI + プロセス） | 1 点訴求のみ、残りは画像 |
| 動画あり（デモ等）| 動画を見ない人にも伝わる summary |

## 例文

### 例 1: understated × oss-dev × 添付なし

> claude-plugins v0.2.0 を公開しました。
> design-director に告知ツイート生成スキル buzzounce を追加して、
> persona と mode を選んで日本語ツイートを対話で組み立てられる
> ようにしています。
>
> https://example.com/your-org/your-repo

### 例 2: understated × oss-dev × 添付 1 枚（hero）

> claude-plugins v0.2.0 を公開しました。
> 公開のたびに「何を書けばいいか」で止まる時間を削りたくて、buzzounce
> という告知ツイート生成スキルを追加しています。
>
> https://example.com/your-org/your-repo

→ 1 枚目に hero 画像があるので、文面はストーリー寄り（"止まる時間を削り
たくて"）。機能詳細は画像に任せる。

### 例 3: casual × general × 添付 1 枚

> v0.2.0 リリースしました 🎉
> 告知ツイート書くのに毎回時間かかるので、persona / mode を選ぶだけで
> 案を出してくれるスキル buzzounce を入れてみました。
>
> https://example.com/your-org/your-repo

### 例 4: formal × oss-dev × 添付なし

> claude-plugins v0.2.0 を公開いたしました。
> 本バージョンでは、告知ツイート生成スキル "buzzounce" を新たに追加して
> おります。persona と mode の指定により、用途に応じた日本語ツイートを
> 対話で組み立てる仕組みです。
>
> https://example.com/your-org/your-repo

## NG 構成

- フックを疑問文で始める（「v0.2.0 リリースしました知ってますか？」）
- バージョン番号だけのフック（「v0.2.0」だけ。何のリリースかわからない）
- 全機能を網羅的に列挙する（読みづらい、3 個までに絞る）
- 「！！！」連発の感嘆
- 過去バージョンへの言及をくどく入れる（読者は文脈持ってない）

## release type で迷ったら

- 1 つの ツイートに **何が新しいか + 1 行 + リンク** だけ残るか確認
- 余分な thoughts / 経緯 / 苦労話は次のツイート（lesson type 単独で別投稿）
  に逃がす
- リンク先で詳細が読めるなら、ツイート側は **トレーラー** に徹する
