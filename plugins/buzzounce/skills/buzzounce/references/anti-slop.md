# anti-slop — 全 mode / 全 persona 共通の NG 表現

ツイートが「自動生成された」「インフルエンサー商法っぽい」「煽り屋っぽい」
と読まれた時点で、OSS / dev コミュニティでは即座にスクロール対象になる。
以下の表現は **mode や persona に関わらず** 避ける。

## カテゴリ別 NG リスト

### 1. AI 調敬語

自動生成バレを起こす定型句:

- 「本日は〜のご紹介です」
- 「お役立ていただければ幸いです」
- 「いかがでしたでしょうか」
- 「最後までお読みいただきありがとうございました」
- 「以下の点に留意してください」（ツイートで使うと教科書感）

### 2. 煽りワード

OSS / dev 規範では浮く強調語:

- 「最強」「神」「ヤバい」「神ツール」「神プラグイン」
- 「革命的」「革新的」（実態を伴わない言葉のインフレ）
- 「知らないと損」「知らないとヤバい」「これ知ってるだけで」
- 「衝撃」「驚愕」「マジで」「ガチで」（連発するとうるさい）
- 「〜すぎる」（"便利すぎる"・"早すぎる" 等の安易な強調）

### 3. 過剰自己宣伝

押しつけがましい印象:

- 「！」の連発（"！！！" や "！？！？"）
- 「ぜひ使ってみてください！」（懇願感 / プレッシャー）
- 「フォロー必須です」「保存推奨」「RT お願いします」（無料の投稿で
  対価を要求する形）
- 「自信作です」「渾身の出来です」（自己評価語）

### 4. インフルエンサー口調

副業 / ライフハック系から漏れ出る独特の語感:

- 「これ、まじでヤバいです」
- 「〜したら人生変わった」「〜で月◯万」
- 「全人類が使うべき」
- 「99% の人が知らない」
- 「結論から言うと、◯◯です。」（YouTube 動画みたいな書き出し）

### 5. 専門用語の連発（説明なし）

ターゲットが dev でも、ジャーゴンを並べると distance が出る:

- 「〜を Tree-sitter で AST に落として LSP と統合しました」
  → 「コード解析ライブラリと組み合わせました」相当に噛み砕く（一回は
  わかる言葉で言い直す）
- 略語連発（"PR / CI / CD / DX" を 1 ツイートに 4 つ等）

### 6. ひけらかし的言い回し

知識マウント / 過剰自慢の匂い:

- 「実は〜なのです（ドヤ）」（"実は" は煽り用法だと NG、純粋な背景説明
  ならOK）
- 「皆さんご存知のように」（読者を試す距離感）
- 「私はこう思います」（強い one-shot 主張形、文脈次第で OK だがツイート
  単独だと角ばる）
- 「〜すべきです」「〜してはいけません」（規範を一方的に押し付ける）

### 7. 嫌な印象を与える表現全般

定型化しにくいが、読み返して以下を感じたら NG:

- マウント（「〜できないのは情弱」「〜知らないとか…」）
- 自慢（「年収が」「フォロワーが」「PV が」を主役に置く）
- 攻撃性（特定の他社 / 他人 / ツールを名指しで貶める）
- 上から目線（「教えてあげます」「分かるかな？」）
- 不安煽り（「これやらないと取り残されますよ」）

## 例: NG → OK

| NG | OK |
|---|---|
| 「神プラグインを公開しました！！！」 | 「プラグインを公開しました。」 |
| 「知らないと損する Claude Code の使い方」 | 「Claude Code でこういう使い方をしています」 |
| 「全人類使うべき」 | 「もし合いそうなら、興味あれば」 |
| 「本日は〜のご紹介です」 | 「〜を作りました」 |
| 「自信作です！」 | （何も書かない、もしくは「気に入ってます」） |

## 判定の最終手段

**読み返して「自分が読み手だったら、この発信者を信頼するか？」と問う。**
NO なら書き直す。
