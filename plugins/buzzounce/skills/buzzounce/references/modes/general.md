# mode: general

副業 / 生活情報 / 一般読者向けの規範。**共感型・有益情報型** が機能する
コミュニティのトーン。OSS / dev コミュニティとは別ルール。

## 想定読者

- AI ツールを業務外で使ってみたい人
- 副業 / 生産性向上に関心がある一般ビジネスパーソン
- 読者層が技術専門でない（コードを日常的に書かない）
- フォローのきっかけが「役立つ情報」「共感」「実体験」

## 絵文字

**1-3 個まで**。0 個も可（persona 側で 0 を選んだ場合）。

- 推奨: 1-2 個（フックや感情を強調する箇所に）
- OK: 3 個（複数項目をリスト化する時の頭文字代わり 等）
- NG: 4 個以上（密度過多、AI 感が出る）
- NG: 関係ない装飾絵文字の多用

例: 🎉（祝）/ 📝（メモ）/ ✨（明るい紹介）/ 💡（気づき）/ 🚀（前進）

## ハッシュタグ

**0-3 個まで**。

- OK: 0 個（持ち味で勝負）
- OK: 1-3 個（カテゴリタグとして）
- NG: 4 個以上（ジャンクっぽくなる）
- NG: 「#OSS」「#エンジニア勉強中」等を oss-dev 文脈で混ぜる（コミュニティ
  ミスマッチ）

## 推奨構成

以下の 4 テンプレ系列が有効:

### ① 共感型「あるある」

- フック: 言語化されてない "あるある" を 1 行
- 展開: 状況を 2-3 行で具体的に
- オチ: 「これ、自分だけ？」「わかる人いる？」

### ② 有益情報型「まとめ」

- フック: 「知ってると◯◯系」（ただし「知らないと損」は煽りすぎなので避け
  て、代わりに「最近やってみて良かった」等に）
- 本文: 箇条書き 3-5 個
- CTA: 保存・シェア促進句（"保存推奨" は angry になりすぎ、"あとで見返す
  と便利かも" 程度に抑える）

### ③ 自己開示型「体験談」

- 過去（ネガティブ） → 転機 → 現在 → メッセージ
- 数値で説得力（"3 時間 → 30 分" 等）

### ④ 問いかけ型「議論喚起」

- フック: 「みんなどうしてる？」系
- 本文: 自分の状況や仮説
- CTA: リプライ募集（"ぜひ教えてください" OK）

## 文字数

- 140-280 字 が目安
- 共感型は **140-200 字**
- 有益情報型は **200-280 字**
- 体験談型は **200-280 字**

## 禁止事項

`anti-slop.md` の全カテゴリは general でも禁止。一方で oss-dev で禁じた
以下は general では **緩和** される:

- 「ぜひ教えてください」「保存しておきたい人いる？」のような **リプライ /
  保存促進** は OK（ただしくどくしない）
- 絵文字使用 OK（1-3 個まで）
- ハッシュタグ使用 OK（0-3 個まで）
- スレッド連投 OK（最大 3 ツイートまで）

ただし以下は general でも依然 NG:

- 「最強」「神」「99% の人が知らない」（インフルエンサー口調）
- 「！！！」連発
- マウント・ひけらかし
- 「月 ◯◯ 万」系の収益マウント

## 例文（general mode の典型）

### 例: 共感型 / casual persona

> AI ツール導入で「これ何に使うんだ？」って一瞬でも止まったとき、
> その「何に使うか」を一緒に考えてくれるツールがあると一気に楽
> なんですよね 🙂
>
> 自分だけかな？

特徴:
- 体験 + 主観（「楽なんですよね」）
- 絵文字 1 個（🙂）
- 「自分だけかな？」で読者参加を促す

### 例: 有益情報型 / casual persona

> Claude Code のプラグイン、最近こうやって使ってます:
>
> ・新規プロジェクト → design-director でデザイン探索
> ・公開時 → buzzounce で告知ツイート対話生成
> ・素材レンダリング → tools/render で PNG / WebM 化
>
> 詳しくはリポへ。気になる方はどうぞ。
> https://example.com/your-org/your-repo

特徴:
- 箇条書き 3 個
- フックなし（情報訴求のみ）
- 「気になる方はどうぞ」で柔らかい CTA

### 例: 体験談型 / casual persona

> 個人 OSS の告知ツイート、毎回考えるのに 30 分かかってたんですが、
> 自分のプラグイン（buzzounce）に書かせるようにしたら 5 分で済む
> ようになりました ✨
>
> 「画像 1 枚目に何があるか」まで聞いてくれるので、画像と文面が
> 重複しなくなったのも大きい。
>
> https://example.com/your-org/your-repo

特徴:
- 過去（30 分）→ 現在（5 分）の構造
- 絵文字 1 個（✨）
- 自作で自作を使う dogfood ストーリーを共感型に寄せた

## このモードで生成するときのチェックリスト（要約）

1. 絵文字 0-3 個
2. ハッシュタグ 0-3 個
3. 「最強」「神」「知らないと損」を使っていない
4. 「！」が 0-1 個（連発しない）
5. リプライ / 保存促進は控えめに（"ぜひ教えてください" 程度）
6. 共感 / 有益情報 / 体験談 / 議論のいずれかの型に沿っているか
