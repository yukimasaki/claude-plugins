# mode: oss-dev

OSS / 個人開発 / AI ツール / dev Twitter 向けの規範。**煽り・宣伝色を
極力排し、技術と craft の話に集中する** コミュニティ規範に従う。

## 想定読者

- Claude Code / Cursor / Copilot 等の AI 開発ツールを使うエンジニア
- 個人開発者 / OSS contributors
- 技術記事を Zenn / Qiita / dev.to で読む層
- ツールやライブラリの「中身」「使い心地」を見る人

## 絵文字

**0-1 個まで**。基本ゼロ。

- OK: ゼロ。最も安全
- OK: 1 個（特定の意味があるとき。例: 🎉 release / 🚧 WIP / 📝 docs）
- NG: 2 個以上
- NG: 装飾目的のスタンプ的絵文字（❤️ ✨ 💡 を文末に飾る等）

## ハッシュタグ

**なし（ゼロ個）**。

- 「#OSS」「#ClaudeCode」「#AI」等は dev Twitter で **ダサ認定** されやすい
- ハッシュタグなしの方が impression が伸びる傾向（直接フォローや RT 経由で
  広がる文化）
- 例外: イベント告知（`#claudecon` 等の特定イベント公式タグ）は OK

## 推奨構成

**フック → 本文 → リンク** の 3 ブロック構成。

### フック（1 行目、〜 50 字推奨）

- **数字** で始める（"3 つの変更"・"30 分かかってた作業が 5 分に"）
- **逆説** で始める（"〜と思ってたけど違った"・"車輪を再発明しました"）
- **具体名** で始める（"claude-plugins v0.2.0 を公開しました"）
- 疑問文や煽りは使わない（「これ知ってますか？」NG）

### 本文（中央、〜 150 字推奨）

- 何が新しいか / なぜ作ったか / どう使うか のいずれか **1 点に絞る**
- 詳細は箇条書きより文章の方が dev Twitter では落ち着く
- 略語は最初に展開してから使う（"Pull Request (PR)" 一回、以降 PR）

### リンク（最終ブロック）

- 通常 1 個まで（複数あると distraction）
- リンク先で何が見られるか一行で示す（"全プロンプトログを公開"）
- リンクの直前に空行を 1 つ（読みやすさ）

### 文字数

- フック: 〜 50 字
- 本文: 〜 150 字
- 全体: **140-280 字** が目安。140 字版と 280 字版の両方を作って選ばせる
  と良い

## 禁止事項

`anti-slop.md` の全カテゴリに加え、oss-dev 固有の地雷:

- **「結論から言うと、」で始まる構成**（YouTube 動画みたい）
- **長文スレッド連投**（dev Twitter では単発の方が好まれる。スレッド
  にする場合も 3 ツイート以内）
- **「#今日の積み上げ」「#エンジニア勉強中」系のハッシュタグ**（副業
  系コミュニティの記号として浮く）
- **「○ ○ で月 ○○ 円」系の収益訴求**
- **誰かを名指しでメンション** して比較する形（"X より Y の方がいい"）

## 例文（oss-dev mode の典型）

### 例: release（understated persona との組み合わせ）

> claude-plugins v0.2.0 を公開しました。
> design-director に告知ツイート生成スキル buzzounce を追加して、
> persona と mode を選んで日本語ツイートを対話で組み立てられる
> ようにしています。
>
> https://example.com/your-org/your-repo

特徴:
- 絵文字なし
- ハッシュタグなし
- フック = 具体名 + 動詞（公開しました）
- 本文 1 文で何が新しいかを伝える
- リンク前に空行

### 例: tip（understated persona との組み合わせ）

> Claude Code のスキルで persona を切り出して別ファイルにしたら、
> 後から「真面目」「カジュアル」を増やすときに SKILL.md を触らずに
> ファイル追加だけで済むようになりました。
> 進化させやすい構造、推せます。

特徴:
- 数字なし、逆説なし、具体例ベースのフック
- 自分の経験から一般化（"進化させやすい構造"）
- 控えめな評価（"推せます"）でちょうど良い

## このモードで生成するときのチェックリスト（要約）

1. 絵文字 0-1 個
2. ハッシュタグゼロ
3. フックが煽りでない（疑問形・"これヤバい" 系を使っていない）
4. リンク先で何が見られるか示せている
5. スレッドにしていない（または 3 ツイート以内）
6. 「結論から言うと」「99% の人が知らない」等の YouTube 調を使っていない
