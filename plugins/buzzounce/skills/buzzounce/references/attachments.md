# attachments — 添付素材を踏まえた文面調整ガイド

X 投稿の多くは画像 / 動画と一緒に出される。文面と素材の関係を意識せずに
書くと、(a) 画像で見えているものを文面でも繰り返す（重複）か、(b) 文面が
素材を全く参照しない（バラバラ）か、どちらかになる。本ガイドは添付素材の
有無と内容に応じて文面の役割を変える指針。

## ブリーフ収集で必ず聞く 3 項目

スキルは step (1) のブリーフ収集で以下を **必ず** ユーザーに確認する:

1. **添付の有無**: 「画像 / 動画を添付する予定はありますか？」（yes / no）
2. **枚数（種類別）**: 画像 N 枚 / 動画 M 本（X の制限上、画像最大 4 枚 +
   動画 1 本まで）
3. **各 N 枚目の順番と内容**: 「1 枚目は何の画像？ 2 枚目は？」と順番に
   聞く

例:

```
1 枚目: ヒーロー画像（design-director で生成、claude-plugins のロゴ的な絵）
2 枚目: ギャラリー UI のスクショ
3 枚目: 制作プロセスの一場面
動画 1 本: 30 秒のデモ（ブリーフ → 3 案 → 採用 → 納品まで）
```

これを skill が把握した上で文面生成・self-check を行う。

## 添付パターン別の文面の役割

### 添付なし

- 文面単独で全部伝える必要
- **URL を強調**（リンク先で何が見えるか一行で示す）
- 視覚的フックを文中に作る（**具体名・固有名詞・数値** で読者の頭に絵を
  描かせる）
- 例: 「2 ステップ」「1280×640 PNG」「3 案並列」など、頭の中で映像化
  できる表現

### 画像 1 枚

- 1 枚目の画像が「何が見えるか」を伝える
- 文面は **「なぜそれを見るべきか / 背景 / ストーリー」** を語る
- 画像説明（"ロゴが〜"・"UI が〜"）を **再描写しない**
- 例: 1 枚目が hero 画像なら、文面は「自分のプラグインに作らせた」という
  process を語る（dogfood 型）。画像のデザイン詳細は説明しない

### 画像 2-4 枚

- 各画像の **繋がり** を意識:
  - before → after（2 枚）
  - 段階的解説（時系列・状態遷移）
  - 比較（A vs B vs C）
  - グリッド展示（4 枚で 1 つのテーマ）
- 文面は **最も語りたい 1 点** に絞る。残りは画像に任せる
- 各画像にコンテキストを与える alt text を別途準備すると親切（投稿時に
  ユーザーが手動で設定）

### 動画あり

- 文面に **「何を見るべきか」のヒント** を書く
  - 例: 「30 秒のデモ。最後の納品までの導線が肝です」
- 動画を見ない人にも伝わる **短い summary を併記**（動画依存の文面 NG）
- 動画は注目度が上がる（X のフィードで自動再生）ので、文面は控えめでも
  通る

## post-type 別の典型添付パターン

各 post-type ファイルで詳述するが、概要は以下:

| post-type | 典型添付 | 文面の役割 |
|---|---|---|
| `release` | 1 枚目 hero / SPI 画像 | 機能訴求 + リンク誘導 |
| `dogfood` | 自作で生成した素材 | process / why のストーリー |
| `tip` | before / after 2 枚 | 数字 or 教訓を short に |
| `show-and-tell` | スクショ 1-4 枚 | 1 点に絞った語り |
| `lesson` | 失敗の証拠画像 / before | 教訓・気づきを強調 |

## self-check で必ず検査する項目（再掲）

`self-check.md` の項目 7「添付素材の内容と文面が重複していない / 補完
関係になっているか」を必ず実行。

具体的なチェック観点:

- 1 枚目に映っているもの（hero / UI / ロゴ等）を文面で **詳細に描写
  していないか**
- 動画で動きが見せられる場合、文面で **動きを再描写していないか**
- 添付なしの場合、文面単独で **何を見るべきかが伝わるか**

## 例: 添付有無での文面の違い（同じ告知）

**告知**: claude-plugins v0.2.0 公開。design-director に告知ツイート生成
スキル buzzounce を追加。

**添付なし版**:

> claude-plugins v0.2.0 を公開しました。
> design-director に告知ツイート生成スキル buzzounce を追加しています。
> persona と mode を選んで、自然な日本語ツイートを対話で組み立てる
> 形のスキルです。
> https://example.com/your-org/your-repo

→ 文面で「何ができるか」を全部書く

**添付 1 枚（hero 画像）版**:

> claude-plugins v0.2.0 を公開しました。
> design-director に告知ツイート生成スキル buzzounce を追加して、
> 公開のたびに「何を書けばいいか」で止まる時間を削っていく試みです。
> https://example.com/your-org/your-repo

→ 文面はストーリー（「公開のたびに止まる時間」）に寄る。画像で機能感は
伝わる前提

**添付 4 枚（hero + persona 選択 UI + 候補 3 案 + 納品テキスト）版**:

> claude-plugins v0.2.0 を公開しました。
> 「ツイート 1 本書くのに 30 分かかる」を解消したくて、persona と mode を
> 選ぶだけで日本語の告知文を対話生成する buzzounce を追加しています。
> https://example.com/your-org/your-repo

→ 文面は問題提起 + 解決のサマリ。ステップ詳細は 4 枚の画像に任せる
