# Pre-flight checklist (internal)

Silently answer before writing HTML. Do NOT print the answers.

1. **Artifact type** — pick one: `landing | case_study | dashboard | pricing | slide | email | one_pager | report`. Two fit? Pick the primary conversion job.
2. **Emotional posture** — confident · playful · serious · friendly · editorial · technical. Show in type weight, palette saturation, spacing — not just copy. If a `design-md/{family}/*.md` is selected, the `description:` field in its frontmatter ends with `Mood — ...` mood words; lift those instead of guessing.
3. **Density target** — list section beats meeting the type's floor before `<body>`.
4. **Comparisons** — if brief has "before/after", "前后", "对比", "vs", "from X to Y", or any growth %, name which sections render side-by-side or paired.
5. **Featured numbers** — each number → big-number block (label + source line), not inline prose.
6. **Palette plan** — bg + surface + text + muted + accent (oklch) + secondary/success, optional gradient. Dark ≠ one black + one accent; add mid-tone surface and warm/cool tilt.
7. **Type ladder** — four steps (display · h1 · body · caption) with weight contrast. Fraunces for editorial / case_study / report; Geist or preferred sans for landing / dashboard / pricing.
8. **Anti-slop guard** — scan for lorem ipsum, generic icon-title-text grids, stock testimonials, single accent on flat black, default Tailwind grays, placeholder.com images, abstract decorative shapes used as filler. Replace before generating.
9. **Rot check (long-lived artifacts only)** — if the artifact is a hero, README graphic, social preview, or other long-shelf-life surface, scan for version numbers, calendar years, ordinals (`first`, `第一弾`), and counted nouns (`9 subcommands`). Neutralize or move to a dynamic source (badge, runtime value) before generating.
10. **CJK typography (Japanese targets only)** — if the brief, audience, or referenced sites are Japanese, verify the page applies the rules in `prompts/japanese-typography.v1.txt`: explicit CJK font fallback chain, body line-height ≥ 1.6, letter-spacing ≥ 0 on CJK runs, `palt` on display, no `word-break: break-all` on body, mixed-language strings handled. Skip this check for non-Japanese artifacts.
11. **JSX/TSX format check (design-director only)** — variations are saved as `.tsx` function components with inline styles and token constants, NOT as `.html` files and NOT as `` const html = `...` ``-string-stuffed `<iframe srcDoc>` wrappers. If you caught yourself rationalizing "HTML is simpler / lighter / faster here" or "I'll JSX-ify it later," redesign the file. The `references/design-skills/*.jsx` templates (landing-page, dashboard, heroes, etc.) are the canonical shape. Skip this check for artifacts outside design-director.

If any answer is "not sure" or "default", redesign it before generating.
